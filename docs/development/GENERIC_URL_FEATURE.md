# 兜底 URL 处理功能

## 功能概述

新增了通用 URL 处理功能，可以处理直接的音视频下载链接，不依赖特定平台的 API。

## 主要特性

### 1. 通用下载器 (GenericDownloader)

- **文件类型检测**: 支持常见的音视频格式 (mp3, mp4, m4a, wav, avi, mkv 等)
- **URL 检测**: 通过文件扩展名和 Content-Type 判断是否为媒体文件
- **兜底处理**: 当没有匹配的平台下载器时自动使用

### 2. 增强的下载功能

- **断点续传**: 支持下载中断后从断点继续
- **自动重试**: 最多重试 3 次，处理网络异常
- **进度显示**: 每 10 秒显示下载进度
- **大文件优化**: 
  - 1MB 块大小下载
  - 30 秒连接超时，300 秒读取超时
  - 对于大文件（>20MB）发送企微进度通知

### 3. 智能标题生成

- **LLM 生成**: 当文件没有标题时，使用大模型根据转录文本生成
- **回退机制**: 如果 LLM 生成失败，使用默认标题"自定义文件总结"
- **长度限制**: 生成的标题不超过 20 个字符

### 4. 缓存跳过机制

- **通用 URL 不缓存**: 确保每次都重新下载和转录
- **平台 URL 仍缓存**: 保持原有的缓存检查逻辑

## 工作流程

```
1. 接收 URL → 2. 创建下载器工厂 → 3. 尝试平台特定下载器
                                    ↓ (无匹配)
4. 使用通用下载器 → 5. 检测媒体文件 → 6. 跳过缓存检查
                                    ↓
7. 下载文件 (支持断点续传) → 8. 转录 → 9. LLM 生成标题 → 10. 发送通知
```

## 配置要求

无需额外配置，使用现有的配置项：
- `llm.summary_model`: 用于生成标题的模型
- `llm.api_key`: LLM API 密钥
- `storage.temp_dir`: 临时文件存储目录

## 支持的文件格式

### 音频格式
- MP3, WAV, M4A, AAC, OGG, FLAC, WMA

### 视频格式  
- MP4, AVI, MKV, MOV, WMV, FLV, WebM, M4V

## 使用示例

### API 调用
```bash
curl -X POST "http://localhost:8000/api/transcribe" \
  -H "Authorization: Bearer your_token" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://example.com/audio.mp3",
    "use_speaker_recognition": false
  }'
```

### 支持的 URL 类型
```
✓ https://example.com/file.mp3
✓ https://cdn.site.com/media/audio.m4a
✓ https://storage.com/videos/clip.mp4
✗ https://example.com/page.html (非媒体文件)
```

## 错误处理

### 下载失败处理
1. **连接错误**: 自动重试，最多 3 次
2. **超时错误**: 增加超时时间，支持断点续传
3. **编码错误**: 检测并重试下载
4. **文件大小不匹配**: 验证完整性，必要时重新下载

### 降级策略
1. **LLM 生成标题失败**: 使用默认标题
2. **企微通知失败**: 不影响主流程
3. **非媒体文件**: 返回明确错误信息

1. **分块下载**: 1MB 块大小，减少内存使用
2. **流式处理**: 边下载边写入，不占用大量内存
3. **断点续传**: 避免重复下载已完成部分
4. **超时优化**: 针对大文件调整超时参数

## 部署注意事项

1. **磁盘空间**: 确保临时目录有足够空间存储大文件
2. **网络稳定性**: 建议在网络稳定的环境中部署
3. **LLM 配置**: 确保 LLM API 配置正确
4. **监控告警**: 关注下载失败和重试日志

## 已知限制

1. **文件大小**: 受磁盘空间和网络超时限制
2. **格式支持**: 仅支持常见的音视频格式
3. **网页解析**: 不支持从网页中提取媒体链接
4. **加密内容**: 不支持需要认证的受保护内容
